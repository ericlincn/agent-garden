# 视觉品味验证规则

供视觉检查工具（如 `image_understand`）判断页面视觉质量时参考。规则按严重程度分为“硬性阻断”和“建议优化”。

## 硬性阻断（出现任意一条即判定为不通过）

1. **CSS 完全丢失**：页面没有任何样式，显示为纯 HTML 结构（无背景、无颜色、无任何布局效果）。
2. **关键样式错乱**：出现明显布局崩坏，例如：
   - 元素重叠或错位导致无法正常点击
   - 内容超出视口宽度（出现横向滚动条，但设计不应有此要求）
   - 导航栏、主要按钮等核心组件不可见或位置错乱
3. **资源加载失败**：大量图片、字体、图标显示为破损占位符，严重影响功能理解或使用（超过页面总资源的 30%）。
4. **移动端适配严重问题**：在典型移动视口宽度（375px 或 768px）下，主要内容被截断、或缩放比例异常（例如文字过小无法阅读，或元素过大超出屏幕）。
5. **可读性极差**：文本与背景之间的对比度低于 WCAG AA 标准（即正常文本对比度 < 4.5:1，大文本 < 3:1）。典型例子：白色背景上的浅灰色字。

## 建议优化（不阻断，但会在报告中提醒）

- 间距不一致：同一页面内相似的区块之间边距不相等，或未对齐到网格基线。
- 颜色搭配不协调：使用了过于刺眼（高饱和、高亮度）的背景色，或前景色与背景色冲突产生视觉疲劳。
- 字体大小不一致：正文、标题、辅助文字的字号缺乏规律（例如正文出现多种字号）。
- 动画卡顿或缺失：过渡效果不流畅，或本应有微交互的地方毫无反馈。

## 评分阈值

`image_understand` 工具应返回 0–1 之间的视觉评分。判断标准如下：

| 评分范围   | 结论                     |
| ---------- | ------------------------ |
| < 0.70     | 不通过                   |
| 0.70–0.85  | 通过但需人工复核         |
| > 0.85     | 良好                     |

## 执行要求

- 对每个关键页面（登录页、主控台、列表页、表单页）**分别**进行评分，并输出每个页面的评分和简要理由。
- 如果页面需要登录态，测试流程必须使用测试账号预置 Cookie（由 `visual-tester` 工具负责准备）。不得因为未登录而误判样式丢失。
